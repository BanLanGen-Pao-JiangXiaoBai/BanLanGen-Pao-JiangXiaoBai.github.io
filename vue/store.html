    # vue中简单Store理解

    # 首先是创建一个全局的store对象使每个组件实例都能够拿到，然后这个Store对象中有state属性，他的值是一个对象，因为js中对象为引用类型，将这个属性注入都每个实例中时，每个实例都指向的是同一个对象（state），以确保他们共享同一个对象，并且每个实例中都将state注册成响应式系统，所以state都引用的是同一个并且也将他注册成响应式的，只要他进行修改，每个实例的view都会更新。store中也可以设置state的赋值和清空方法，记录相应的日志。但是如果state并不是引用类型，是值传递，修改state并不会更新视图。

        <div id="app">
        {{mes.message}}
        <button @click='change'>change</button>
    </div>
    <div id="app1">
        {{mes.message}}
        <button @click='change'>change</button>
    </div>
    <script>
        var store={
                state: {
                    message: 'Hello!'
                },
            setMessageAction (newValue) {
                console.log(new Date()+'修改')
                this.state.message = newValue
            },
            clearMessageAction () {
                console.log(new Date()+'清空')
                this.state.message = ''
            }
        }
        var vm=new Vue({
            el:'#app',
            data() {
                return {
                    mes:store.state
                }
            },
            methods: {
                change(){
                    store.setMessageAction('app改变')
                }
            },
        })
        var vm1=new Vue({
            el:'#app1',
            data() {
                return {
                    mes:store.state
                }
            },
            methods: {
                change(){
                    store.setMessageAction('app1改变')
                }
            },
        })
    </script>

    # 在Vuex中state中定义的对象是被冻结的，不能修改其观察者get、set，所以将他赋值给实例data中是不会有响应的，因为其已经被冻结，是不能再给他注册观察者的，注意state对象并没有冻结。
