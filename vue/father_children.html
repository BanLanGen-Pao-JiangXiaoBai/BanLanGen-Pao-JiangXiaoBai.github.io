    # 组件传参

    # 通过slot插槽传参

    # 子组件内通过给slot绑定插槽prop抛出一个属性，父组件通过v-slot='插槽作用域对象'接收这个属性，然后通过这个对象进行访问属性，也可以通过解构的方式直接拿到这个抛出的属性
        <div id="app">
        <banner>
            <template v-slot:banner='slotScope'>
                {{slotScope.banner.x}}&&{{slotScope.banner.y}}
            </template>
        </banner>
    </div>
    <script>
        Vue.component('banner',{
            template:"<div>我是banner组件<slot name='banner' :banner='banner'></slot></div>",
            data() {
                return {
                    banner:{
                        x:1,
                        y:2
                    }
                }
            },
        })
        var vm=new Vue({
            el:'#app',
        })
    </script>

    # 通过eventBus

    # 新创建一个vue实例对象，通过$on('自定义事件',function(){...})监听自定义事件，通过引入该实例或者注入主实例中，在需要调用的地方用那个新创建的实例对象调用$emit('自定义事件名','参数')触发这个事件。主要用来组件之间传参
        <div id="app">
        <demo1></demo1>
        <demo2 ></demo2>
    </div>
    <script>
        var eventBus=new Vue({

        })
        Vue.component('demo1',{
            template:"<div>我是demo1<button @click='demo1'>demo1</button></div>",
            data(){
                return{
                    demo1msg:'我是demo1中的数据'
                }
            },
            methods:{
                demo1(){
                    // 触发eventbus事件并传递过去参数
                    eventBus.$emit('eventbus',this.demo1msg)
                }
            }
        })
        Vue.component('demo2',{
            template:"<div>我是demo2,接收demo1传过来的数据:<span>{{demo2msg}}</span></div>",
            data(){
                return{
                    demo2msg:'我是demo2的数据'
                }
            },
            created(){
                // 监听eventBus中的eventbus事件,箭头函数使this指向demo2组件,修改组件内的数据
                eventBus.$on('eventbus',(v)=>{
                    this.demo2msg=v
                })
            }
        })
        var vm=new Vue({
            el:'#app',
        })
    </script>

    # $ref

    # 通过给子组件或者元素设置ref='名字',父组件通过this.$refs.名字访问到子组件或者子元素,只有在当前文件中ref设置了才能访问到，父组件中给孙组件定义了ref,子组件中是不能访问到的，只有父组件可以访问到

    # $refs 只会在组件渲染完成之后生效，并且它们不是响应式的。这仅作为一个用于直接操作子组件的“逃生舱”——你应该避免在模板或计算属性中访问 $refs。

    # provide注入与inject接收

    # 然而，依赖注入还是有负面影响的。它将你应用程序中的组件与它们当前的组织方式耦合起来，使重构变得更加困难。同时所提供的 property 是非响应式的。这是出于设计的考虑，因为使用它们来创建一个中心化规模化的数据跟使用 $root做这件事都是不够好的。如果你想要共享的这个 property 是你的应用特有的，而不是通用化的，或者如果你想在祖先组件中更新所提供的数据，那么这意味着你可能需要换用一个像 Vuex 这样真正的状态管理方案了。
