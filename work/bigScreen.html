# 拖拽和布局插件 vue-grid-layout

# 可以实现拖拽与栅格布局，拖拽功能上可配置性不高，一个元素移到另一个元素上会把另一个元素挤走，想进行排列的时候这个挤走元素的功能相当恶心，并且是不可配置的，要想解决这个需要在一个元素拖拽的时候把其他元素的位置保存起来，当这个元素拖拽完成时将其他元素移回保存的位置。

# 栅格布局上比起element大小比较精确，但是比较繁琐，不太好用，需要定义好每一个grid-item的x,y,w,h,i值，然后循环出每一个栅格，每个栅格中的组件(.vue文件)也是循环出来的，而不是自己放入子组件标签，这样就只能用实现。

# 的实现 有俩种方式，一种是vue的仅运行时模式，需要把子组件注册成全局组件，一种是运行时+编译器，这种模式体积要比仅运行时大30%，官方不推荐。

# 仅运行时模式不能使用template属性,否则报错，运行时+编译器可以使用

// 仅运行时模式 需要引入.vue文件将组件注册全局,不能使用对象方式的template属性否则报错
import xxx from 'xx'
Vue.component('xxx',xxx)

<component :is="xxx" />

// 运行时+编译器
const xxx={
    template:`<div> 运行时+编译器 </div>`
}
const yyy={
    template: ()=>import('.vue后缀的文件')
}

<component :is="xxx" />
<component :is="yyy" />
